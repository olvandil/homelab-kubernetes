# Grafana

## Overview

Grafana provides visualisation, dashboards and alerting for the homelab Kubernetes platform.

It is deployed via Helm, rendered using Kustomize with Helm support, and managed through ArgoCD using GitOps.

Grafana uses Longhorn for persistent storage and is exposed externally through Traefik with TLS certificates managed by cert-manager.

---

## Deployment

This application is managed by the ApplicationSet under `applications/`.

The Grafana Helm chart is rendered using Kustomize:

- Helm chart: Grafana Community
- Configuration: `values.yaml`
- Ingress: `ingress.yaml`

No Deployment, Service or PVC manifests are maintained manually, as these are generated by the Helm chart.

---

## Components

| File | Purpose |
|------|---------|
| `kustomization.yaml` | Renders the Grafana Helm chart and includes local resources |
| `values.yaml` | Helm values used to configure Grafana |
| `ingress.yaml` | Exposes Grafana through Traefik with TLS |
| `README.md` | Documentation |

---

## Storage

Grafana stores dashboards, users and configuration on a Longhorn-backed Persistent Volume.

Storage Class:

- `longhorn`

---

## Authentication

The administrator credentials are stored in a Kubernetes Secret.

Current deployment uses:

- `grafana-admin-secret`

This secret is currently created manually and is planned to be migrated to SOPS-managed encrypted secrets.

---

## Access

Internal URL:

- https://grafana.watworld.uk

DNS:

- Public DNS managed by ExternalDNS / Cloudflare
- Internal DNS currently provided by an Unbound override

TLS:

- cert-manager
- Let's Encrypt (Cloudflare DNS-01)
